### US-012: Upload component with per-file enrichment status
Status: in_progress

- [ ] client/src/components/UploadModal.jsx renders as a fixed full-screen overlay when isOpen is true
- [ ] Overlay same style as AssetDetailModal. Header: 'Upload Assets' title, × close button. ESC closes modal

- [ ] On enrich success: file status → 'Done ✓' green
- [ ] On enrich 503: file status → amber 'Uploaded — enrichment unavailable'
- [ ] When ALL files reach a terminal status (done/error/unavailable): 'View in library' button appears. On click: close modal, call onUploadComplete() which triggers getAssets() refresh in BrowsePage
- [ ] Vitest test: dropping a valid JPEG sets its status to 'Pending' in the file list
- [ ] Vitest test: file > 20MB shows 'Too large' status and is not in the pending list
- [ ] Vitest test: Submit button is disabled when file list is empty
- [ ] Vitest test: Submit button label shows correct file count 'Upload 3 files'
- [ ] Vitest test: after mocked uploadFiles + enrichAsset resolve, file status shows 'Done'
- [ ] Vitest test: 'View in library' button appears after all files are done
- [ ] Negative case: uploadFiles rejects for one file → that file shows 'Upload failed' red, other files continue processing independently
- [ ] Negative case: enrichAsset returns 503 → file shows amber 'Uploaded — enrichment unavailable', 'View in library' still appears when all files settle
