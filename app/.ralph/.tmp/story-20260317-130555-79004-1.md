### US-001: Bulk enrichment CLI script
Status: in_progress
Depends on: None

Description:
As a developer, I want a re-runnable CLI script that enriches all unenriched assets with OpenAI Vision so the full 49-asset library has AI-generated metadata.

Acceptance Criteria:
- [ ] File created at server/scripts/enrich-all.js — executable via `node server/scripts/enrich-all.js` from the app/ root
- [ ] Script loads .env from app root using dotenv (require('dotenv').config({ path: path.resolve(__dirname, '../../.env') }))
- [ ] Script opens dam.sqlite using better-sqlite3 at path.resolve(__dirname, '../../dam.sqlite')
- [ ] Queries SELECT * FROM assets WHERE enrichment_source IS NULL to get unenriched assets
- [ ] For each asset: (1) tries to fetch image from Scene7 CDN URL as base64; (2) if fetch fails or no scene7_file, falls back to reading thumbnail file from DATA_DIR (default ../Data/royal relative to server/)
- [ ] If neither image source is available, logs 'SKIP <filename>: no image source' and continues
- [ ] Sends image to OpenAI GPT-4o Vision with full prompt (title, description, tags, location, scene, subjects, mood, channel_hint, confidence)
- [ ] On OpenAI error: retries once with simplified prompt (title, description, tags, location, scene, channel_hint, confidence only)
- [ ] On second failure: logs 'FAILED <filename>: <error message>' and continues to next asset
- [ ] On success: updates assets row with enriched_title, enriched_description, enriched_tags (JSON), enriched_location, enriched_scene, enriched_subjects (JSON), enriched_mood, enriched_channel, enrichment_source='ai-vision', enrichment_confidence
- [ ] After each successful update, syncs FTS5: DELETE FROM assets_fts WHERE id=? then INSERT INTO assets_fts(id,title,description,tags,location,creator,category,subcategory,filename) VALUES(...)
- [ ] Waits ENRICH_DELAY_MS (default 500) between each asset
- [ ] Logs progress: '[X/49] Enriching <filename>... done' or '... retrying...' or '... failed' or '... skipped'
- [ ] Prints summary on completion: 'Done. Enriched: X | Failed: Y | Skipped: Z'
- [ ] Example: running script with valid OPENAI_API_KEY processes all 49 assets and prints summary
- [ ] Negative case: if OPENAI_API_KEY is missing, script logs 'Error: OPENAI_API_KEY not set' and exits with code 1 immediately
- [ ] Negative case: re-running script after all assets are enriched logs 'All assets already enriched. Nothing to do.' and exits 0
