### US-002: Expand /api/filters and /api/assets for channel and scene
Status: in_progress
Depends on: US-001

Description:
As a frontend developer, I want the filters and assets API endpoints to support channel and scene so the UI can filter by AI-enriched dimensions.

Acceptance Criteria:
- [ ] GET /api/filters response includes `channels`: array of distinct non-null enriched_channel values sorted alphabetically
- [ ] GET /api/filters response includes `scenes`: array of distinct non-null enriched_scene values sorted alphabetically
- [ ] GET /api/assets accepts `channel` query param — adds WHERE enriched_channel = :channel to query
- [ ] GET /api/assets accepts `scene` query param — adds WHERE enriched_scene = :scene to query
- [ ] Both new filters compose correctly with existing filters (category, rights_status, q, etc.)
- [ ] Supertest test: GET /api/filters returns channels and scenes as arrays (may be empty before enrichment)
- [ ] Supertest test: after inserting a test asset with enriched_channel='hero', GET /api/filters includes 'hero' in channels
- [ ] Supertest test: GET /api/assets?channel=hero returns only assets with enriched_channel='hero'
- [ ] Supertest test: GET /api/assets?scene=ocean returns only assets with enriched_scene='ocean'
- [ ] Negative case: GET /api/assets?channel=nonexistent returns { total: 0, assets: [] } with 200 status
