### US-010: Search and browse page
Status: in_progress
Depends on: US-009

Description:
As a content author, I want to search and browse assets on the main page with live filters so I can find the right image quickly.

Acceptance Criteria:
- [ ] client/src/pages/BrowsePage.jsx is the main page rendered at route /
- [ ] client/src/components/SearchBar.jsx: text input full-width, placeholder 'Search by keyword, location, mood, ship...', onChange updates q state, debounced 300ms using setTimeout/clearTimeout in useEffect
- [ ] client/src/components/FilterSidebar.jsx: 260px fixed left panel. Sections (each collapsible with a toggle arrow): Category (ships/promotions chips from getFilters()), Subcategory (dynamic chips under selected category), Rights Status (owned/unclear/none chips), Location (select dropdown from getFilters()), Metadata Quality (toggle chips: 'Missing Title', 'Missing Rights', 'Release Placeholder')
- [ ] Active filters shown as removable chips in a row below SearchBar. Each chip has an × button that clears that filter
- [ ] All filter state managed in BrowsePage, passed to getAssets() as params on any change
- [ ] client/src/components/AssetGrid.jsx: CSS grid, repeat(auto-fill, minmax(200px, 1fr)), gap 16px
- [ ] client/src/components/AssetCard.jsx: white card, 8px border-radius, hover shadow. Top: aspect-ratio 16/9 img with object-fit cover (grey #E8E8E8 bg if no thumbnail). Body: 10px padding, 2-line clamped display_title (13px bold #1A1A2E), badges row showing category pill and rights dot, amber ⚠ icon if quality_issues.length > 0
- [ ] Results count label above grid: 'Showing {assets.length} of {total} assets' in grey 12px
- [ ] Loading state: show 12 SkeletonCard components (grey animated shimmer matching card shape) while API call is in-flight
- [ ] Empty state: centred message 'No assets match your search.' with a 'Clear all filters' button that resets all filter state
- [ ] Clicking an AssetCard calls onSelectAsset(asset.id) which is wired to open AssetDetailModal
- [ ] Vitest test: BrowsePage renders 3 AssetCards when getAssets mock returns 3 assets
- [ ] Vitest test: typing 'sunset' in SearchBar triggers getAssets with q='sunset' after debounce (use fake timers)
- [ ] Vitest test: clicking category chip 'ships' calls getAssets with category='ships'
- [ ] Vitest test: clicking active filter chip × removes it and calls getAssets without that param
- [ ] Vitest test: shows 12 skeleton cards while loading is true
- [ ] Negative case: getAssets rejects → error banner 'Failed to load assets. Please try again.' shown above grid, no crash
