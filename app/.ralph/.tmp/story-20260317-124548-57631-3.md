### US-003: Update AssetCard to load thumbnails from media route
Status: in_progress
Depends on: US-002

Description:
As a content author, I want asset cards in the browse grid to display the actual thumbnail image so I can visually identify assets.

Acceptance Criteria:
- [ ] In client/src/components/AssetCard.jsx (or wherever the thumbnail img src is set), update src to: asset.thumbnail_path ? `http://localhost:3001/api/assets/media/${asset.thumbnail_path}` : null
- [ ] If thumbnail_path is null, render a placeholder div (grey background with filename text) — do not render a broken <img>
- [ ] Vitest test: AssetCard with thumbnail_path renders an <img> with src containing '/api/assets/media/'
- [ ] Vitest test: AssetCard with thumbnail_path=null renders a placeholder element, not an <img>
- [ ] Negative case: if the media URL returns a 404 (image not found on disk), the img onError handler swaps to the placeholder — no broken image icon shown
