### US-005: GET /api/assets/:id — asset detail endpoint
Status: in_progress
Depends on: US-004

Description:
As a content author, I want full metadata for a single asset including quality analysis and CDN URL so I can review and use the asset.

Acceptance Criteria:
- [ ] GET /api/assets/:id returns 200 with the full assets row plus computed fields
- [ ] Response includes cdn_url computed from scene7_domain + '/is/image/' + scene7_file (null if either missing)
- [ ] Response includes quality_issues: array of strings for each issue present — 'missing_title' if both original_title and enriched_title are null/empty; 'missing_description' if both are null/empty; 'missing_rights' if enriched_rights_status = 'none'; 'missing_location' if both location fields are null/empty; 'title_equals_description' if original_title == original_description (non-null); 's7_sync_error' if s7_sync_error = 1; 'release_placeholder' if has_release_placeholder = 1
- [ ] Response includes display_title, display_description, display_tags computed fields
- [ ] Supertest test: GET /api/assets/anthem-of-the-seas-new-york-statue-liberty.jpg returns correct original_title
- [ ] Supertest test: an asset with enriched_rights_status='none' has 'missing_rights' in quality_issues
- [ ] Supertest test: an asset with has_release_placeholder=1 has 'release_placeholder' in quality_issues
- [ ] Supertest test: cdn_url is a correctly formatted string for an asset with scene7_file set
- [ ] Negative case: GET /api/assets/does-not-exist returns 404 { error: 'Asset not found' }
