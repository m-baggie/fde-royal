### US-004: Asset card hover states and action overlay
Status: in_progress
Depends on: US-003

Description:
As a content author, I want to hover a card and see quick download and copy actions so I can act without opening the detail modal.

Acceptance Criteria:
- [ ] Card wrapper gets transition: 'transform 200ms ease, box-shadow 200ms ease'
- [ ] On hover: transform translateY(-2px), box-shadow '0 8px 24px rgba(0,0,0,0.12)'
- [ ] A hover overlay div appears at the bottom of the image zone (position absolute, bottom 0, left 0, right 0)
- [ ] Overlay background: rgba(0,0,0,0.6), padding 8px 10px, display flex, gap 8px, justify-content flex-end
- [ ] Overlay entrance: opacity 0 at rest → opacity 1 on card hover, transition 150ms ease
- [ ] Overlay contains a Download button: rendered as <a> tag with href={getAssetDownloadUrl(asset.id)} and download attribute, styled as icon button (↓ symbol), color white, font-size 14px, background rgba(255,255,255,0.15), border-radius 6px, padding 4px 8px, no underline
- [ ] Overlay contains a Copy CDN button: only rendered when asset.cdn_url is present. On click: copies cdn_url to clipboard, button content swaps to ✓ for 2000ms then reverts to ⎘ symbol
- [ ] Card thumbnail image container must have position: relative to contain the overlay
- [ ] Example: hovering a card with a cdn_url shows both Download and Copy CDN buttons in the overlay
- [ ] Negative case: hovering a card with no cdn_url shows only the Download button — no Copy CDN button
- [ ] npm run lint passes
- [ ] npm run build passes
