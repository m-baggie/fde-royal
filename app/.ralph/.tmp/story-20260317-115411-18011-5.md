### US-012: Upload component with per-file enrichment status
Status: in_progress
Depends on: US-009

Description:
As a content author, I want to drag-and-drop or browse for images and have them automatically enriched with AI metadata so new assets are immediately usable in the library.

Acceptance Criteria:
- [ ] client/src/components/UploadModal.jsx renders as a fixed full-screen overlay when isOpen is true
- [ ] Overlay same style as AssetDetailModal. Header: 'Upload Assets' title, × close button. ESC closes modal
- [ ] Drag-and-drop zone: centred box, dashed 2px border #C8960C, border-radius 8px, 200px min-height, text 'Drop images here or click to browse'. On dragover: border becomes solid, bg tints gold lightly. On drop: captures files
- [ ] Hidden <input type='file' accept='image/jpeg,image/png,image/webp' multiple ref> triggered when zone is clicked
- [ ] Toggle button below zone: 'Upload folder' — switches input to include webkitdirectory and multiple attributes. Toggle label changes to 'Upload files' when folder mode is active
- [ ] After file selection, file list renders below zone. Each row: filename (truncated 40 chars), formatted file size, status badge
- [ ] Client-side validation before adding to list: file > 20MB → status 'Too large (max 20MB)' red, excluded from upload. Non-image MIME → status 'Unsupported type' red, excluded. Valid files → status 'Pending' grey
- [ ] Submit button 'Upload {N} files' at bottom of modal — disabled if no valid (pending) files
- [ ] On submit: build FormData appending all valid files under field 'files', call uploadFiles(formData). Each file status → 'Uploading...' blue
- [ ] On upload success: for each { id } in response.assets, immediately call enrichAsset(id). File status → 'Enriching...' purple
- [ ] On enrich success: file status → 'Done ✓' green
- [ ] On enrich 503: file status → amber 'Uploaded — enrichment unavailable'
- [ ] When ALL files reach a terminal status (done/error/unavailable): 'View in library' button appears. On click: close modal, call onUploadComplete() which triggers getAssets() refresh in BrowsePage
- [ ] Vitest test: dropping a valid JPEG sets its status to 'Pending' in the file list
- [ ] Vitest test: file > 20MB shows 'Too large' status and is not in the pending list
- [ ] Vitest test: Submit button is disabled when file list is empty
- [ ] Vitest test: Submit button label shows correct file count 'Upload 3 files'
- [ ] Vitest test: after mocked uploadFiles + enrichAsset resolve, file status shows 'Done'
- [ ] Vitest test: 'View in library' button appears after all files are done
- [ ] Negative case: uploadFiles rejects for one file → that file shows 'Upload failed' red, other files continue processing independently
- [ ] Negative case: enrichAsset returns 503 → file shows amber 'Uploaded — enrichment unavailable', 'View in library' still appears when all files settle
