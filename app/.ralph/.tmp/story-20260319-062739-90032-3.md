### US-003: Header favourites icon with count badge and dropdown tray
Status: in_progress
Depends on: US-001

Description:
As a content author, I want to see my saved favourites at a glance and manage them from the header so I can quickly review my shortlist.

Acceptance Criteria:
- [ ] Header receives count and onFavouriteIconClick props
- [ ] A favourites button is rendered to the right of the Upload button (always visible)
- [ ] Button: 36×36px circular, background transparent, border none, cursor pointer, contains ♥ icon (20px, color #001B6B)
- [ ] Count badge: shown when count > 0. Navy circle (#001B6B), 16px diameter, white text 10px/700, position absolute top-right of the icon
- [ ] A FavouritesDropdown component renders below the icon when open
- [ ] Dropdown: 320px wide, max-height 480px, scrollbar hidden, background #FFFFFF, border-radius 12px, box-shadow as per uiNotes, position absolute top 48px right 0, z-index 200
- [ ] Dropdown header: 'Favourites' label left, count right, padding 14px 16px, border-bottom
- [ ] Dropdown items: read from localStorage 'dam_favourites_data'. Each item shows 44×44px thumbnail, display_title (2-line clamp), × remove button
- [ ] × remove button calls onFavouriteToggle(id) and updates display immediately
- [ ] Dropdown footer (count > 0): 'Clear all' button left, 'Export CDN URLs' button right
- [ ] 'Export CDN URLs': collects cdn_url from stored data (skip nulls), joins with newline, copies to clipboard, button shows '✓ Copied!' for 2000ms
- [ ] 'Clear all': calls clear() and closes dropdown
- [ ] Empty state: shows heart icon and 'No favourites yet' centered
- [ ] Clicking outside the dropdown closes it (useEffect with mousedown listener)
- [ ] Example: 3 assets favourited → badge shows '3', dropdown lists all 3 with thumbnails
- [ ] Negative case: asset with no cdn_url is listed in dropdown but skipped in export
- [ ] npm run lint passes
- [ ] npm run build passes
