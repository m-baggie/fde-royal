### US-009: App shell, layout, and API client
Status: in_progress
Depends on: None

Description:
As a developer, I want a branded app shell with a consistent layout and a centralised API client so all pages share structure and API calls are in one place.

Acceptance Criteria:
- [ ] client/src/components/Header.jsx: fixed header 64px tall, navy bg (#001B6B), 3px gold bottom border. Left side: 'Royal Caribbean' white bold 20px, below it 'DAM Asset Intelligence' in gold 12px. Right side: Upload button in gold (#C8960C), white text, 10px 20px padding, border-radius 6px
- [ ] client/src/App.jsx renders <Header onUploadClick={...} /> above a <main> content area with padding-top 64px to clear fixed header
- [ ] client/src/api/assets.js exports: getAssets(params), getAsset(id), getFilters(), uploadFiles(formData), enrichAsset(id)
- [ ] getAssets(params): GET /api/assets with params serialised as URLSearchParams, returns { total, assets }
- [ ] getAsset(id): GET /api/assets/:id, returns full asset object
- [ ] getFilters(): GET /api/filters, returns { categories, subcategories, locations, rights_statuses, counts }
- [ ] uploadFiles(formData): POST /api/assets/upload with formData, returns { uploaded, assets }
- [ ] enrichAsset(id): POST /api/assets/:id/enrich, returns updated asset object
- [ ] All functions use axios instance with baseURL: 'http://localhost:3001'
- [ ] Vitest test: Header renders text 'Royal Caribbean'
- [ ] Vitest test: Header renders text 'DAM Asset Intelligence'
- [ ] Vitest test: Header renders a button with text 'Upload'
- [ ] Vitest test: getAssets called with { category: 'ships' } makes GET to /api/assets?category=ships (mock axios)
- [ ] Negative case: axios network error propagates — callers handle it, app does not crash
