### US-004: GET /api/assets — search and filter endpoint
Status: in_progress
Depends on: None

Description:
As a content author, I want to search and filter assets via the API so I can find relevant images for my campaign.

Acceptance Criteria:
- [ ] server/src/routes/assets.js created — exports an Express Router
- [ ] server/src/services/search.js exports searchAssets(db, params) function
- [ ] GET /api/assets with no params returns 200 { total: 49, assets: [...] }
- [ ] Query param q: runs SELECT from assets JOIN assets_fts WHERE assets_fts MATCH ? — returns ranked results by FTS rank
- [ ] Query param category: adds WHERE category = ? to query
- [ ] Query param subcategory: adds WHERE subcategory = ? to query
- [ ] Query param rights_status: filters on enriched_rights_status column (values: owned, unclear, none)
- [ ] Query param location: filters on enriched_location or original_location
- [ ] Query param has_title=true: WHERE (original_title IS NOT NULL AND original_title != '') OR (enriched_title IS NOT NULL AND enriched_title != '')
- [ ] Query param has_rights=true: WHERE enriched_rights_status != 'none'
- [ ] Query param has_release_placeholder=true: WHERE has_release_placeholder = 1
- [ ] Query params limit (default 100, max 200) and offset (default 0) for pagination
- [ ] Each asset in response includes: id, filename, category, subcategory, display_title, display_description, display_tags, thumbnail_path, web_image_path, enriched_rights_status, enrichment_source, width, height, scene7_file, scene7_domain, s7_sync_error, needs_rights_review, has_release_placeholder
- [ ] Route registered in server/src/index.js: app.use('/api/assets', assetsRouter)
- [ ] Supertest test: GET /api/assets returns 200 with total 49
- [ ] Supertest test: GET /api/assets?q=sunset returns at least 1 result
- [ ] Supertest test: GET /api/assets?category=ships returns only assets with category='ships'
- [ ] Supertest test: GET /api/assets?rights_status=none returns exactly 27 assets
- [ ] Supertest test: GET /api/assets?limit=5&offset=0 returns assets.length=5 and total=49
- [ ] Negative case: GET /api/assets?q=zzznomatch returns 200 { total: 0, assets: [] }
