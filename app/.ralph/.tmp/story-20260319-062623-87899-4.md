### US-003: Metadata toggle — clean Enriched / Original pill toggle (one view at a time)
Status: in_progress
Depends on: None

Description:
As a content author, I want to switch between enriched and original metadata views cleanly so I'm not overwhelmed by a two-column comparison table.

Acceptance Criteria:
- [ ] Replace the current 'Show Original' button with a two-option pill toggle: left='Enriched', right='Original'
- [ ] Toggle container: border-radius 100px, border '1.5px solid #001B6B', display inline-flex, height 28px
- [ ] Each option: 50% width, font 12px/600, cursor pointer, border-radius 100px
- [ ] Active option: backgroundColor '#001B6B', color '#FFFFFF'
- [ ] Inactive option: backgroundColor '#FFFFFF', color '#001B6B'
- [ ] Default active: Enriched
- [ ] Enriched mode: single 'Value' column. Shows enrichedVal if present, falls back to origVal, falls back to grey italic 'Not enriched'
- [ ] Original mode: single 'Value' column. Shows origVal only. If null/empty shows grey '—'. Does NOT show enriched values
- [ ] No side-by-side layout, no Original column, no gold borders in either mode
- [ ] Toggle state resets to 'enriched' each time the modal opens
- [ ] Example: opening modal → Enriched tab active, table shows enriched values
- [ ] Example: clicking Original tab → table shows only original values, enriched values not visible
- [ ] Negative case: asset with no enriched data → Enriched mode shows original values as fallback; Original mode shows same original values
- [ ] npm run lint passes
- [ ] npm run build passes
