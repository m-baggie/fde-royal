### US-001: Global design tokens and page background
Status: in_progress
Depends on: None

Description:
As a developer, I want shared color and spacing constants defined in one place so all redesign stories use consistent values without scattered hardcoded hex codes.

Acceptance Criteria:
- [ ] A constants file client/src/styles/tokens.js exports: NAVY, GOLD, SURFACE, BORDER, TEXT_PRIMARY, TEXT_SECONDARY, TEXT_MUTED as named string exports
- [ ] BrowsePage.jsx imports SURFACE and applies it as the background color of the main content area and sidebar container (replacing any #fff or white backgrounds on those containers)
- [ ] The sidebar container background is #F7F8FB (#SURFACE)
- [ ] The main grid container background is #F7F8FB (#SURFACE)
- [ ] Cards themselves remain white (#FFFFFF) — only the page/container background changes
- [ ] Example: visiting the browse page shows a subtle off-white page background with white cards sitting on top
- [ ] Negative case: no other component colors are changed in this story — only page/sidebar container backgrounds
- [ ] npm run lint passes
- [ ] npm run build passes
