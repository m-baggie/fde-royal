### US-006: GET /api/filters — filter options endpoint
Status: in_progress
Depends on: US-004

Description:
As a frontend developer, I want a filters endpoint that returns all distinct filter values and counts so the UI can populate filters dynamically.

Acceptance Criteria:
- [ ] server/src/routes/filters.js created — exports an Express Router
- [ ] GET /api/filters returns 200 with { categories, subcategories, locations, rights_statuses, counts }
- [ ] categories: sorted array of distinct non-null category values
- [ ] subcategories: object keyed by category, each value is sorted array of subcategory strings e.g. { ships: ['allure','anthem','grandeur'], promotions: ['asia','casino','star-and-icon'] }
- [ ] locations: sorted array of distinct non-null values from both original_location and enriched_location, merged and deduplicated
- [ ] rights_statuses: array of { value, count } objects for 'owned', 'unclear', 'none'
- [ ] counts: { total, missing_title, missing_description, missing_rights, missing_location, has_release_placeholder, s7_sync_error } — each is an integer
- [ ] Route registered in server/src/index.js: app.use('/api/filters', filtersRouter)
- [ ] Supertest test: categories contains 'ships' and 'promotions'
- [ ] Supertest test: subcategories.ships contains 'allure', 'anthem', 'grandeur'
- [ ] Supertest test: counts.has_release_placeholder equals 7
- [ ] Supertest test: counts.total equals 49
- [ ] Negative case: empty database returns all arrays empty and all counts 0
