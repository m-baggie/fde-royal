### US-004: API and filters — expose destination_region and content_type
Status: in_progress
Depends on: US-001

Description:
As a content author, I want to filter by destination region and content type so I can quickly find the right assets for my campaign.

Acceptance Criteria:
- [ ] GET /api/filters returns destination_regions: string[] (sorted distinct non-null enriched_destination_region values)
- [ ] GET /api/filters returns content_types: string[] (sorted distinct non-null enriched_content_type values)
- [ ] GET /api/assets?destination_region=Caribbean returns only assets with enriched_destination_region='Caribbean'
- [ ] GET /api/assets?content_type=ship-exterior returns only assets with enriched_content_type='ship-exterior'
- [ ] searchAssets() in search.js handles destination_region and content_type filter params
- [ ] FilterSidebar renders a 'Destination Region' chip section and a 'Content Type' chip section using the new filter values
- [ ] AssetDetailModal METADATA_ROWS includes Destination Region and Content Type rows
- [ ] Supertest test: GET /api/filters returns destination_regions array
- [ ] Supertest test: GET /api/assets?destination_region=Caribbean returns subset of assets
