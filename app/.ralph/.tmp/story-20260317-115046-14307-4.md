### US-007: POST /api/assets/upload — multi-file upload endpoint
Status: in_progress
Depends on: US-004

Description:
As a content author, I want to upload image files so they are saved to disk and registered in the database ready for enrichment.

Acceptance Criteria:
- [ ] server/src/routes/upload.js created — exports an Express Router
- [ ] POST /api/assets/upload accepts multipart/form-data with field name 'files' using multer().array('files')
- [ ] multer configured with: storage = diskStorage saving to uploads/ with filename <uuid>-<originalname>, fileFilter rejecting non-image MIME types with error 'Unsupported file type. Accepted: jpeg, png, webp', limits.fileSize = 20 * 1024 * 1024
- [ ] For each accepted file: generate uuid v4 as id, insert asset row with id, filename=originalname, filepath=uploads/<uuid>-<originalname>, mime_type, file_size=bytes, category='uploaded', enrichment_source='pending', created_at=now
- [ ] Response 201: { uploaded: N, assets: [{ id, filename, filepath }] }
- [ ] Route registered in server/src/index.js: app.use('/api/assets', uploadRouter) — note: upload route is POST /api/assets/upload so it must be registered before the :id param route
- [ ] Supertest test: POST with a valid JPEG buffer returns 201 with an asset id
- [ ] Supertest test: uploaded asset is retrievable via GET /api/assets/:id
- [ ] Supertest test: file exists on disk at the filepath in the response
- [ ] Negative case: POST with a .txt file returns 400 { error: 'Unsupported file type. Accepted: jpeg, png, webp' }
- [ ] Negative case: POST with no files field returns 400 { error: 'No files provided' }
- [ ] Negative case: multer fileSize error returns 413 { error: 'File too large. Maximum size is 20MB' }
