### US-008: POST /api/assets/:id/enrich — OpenAI Vision enrichment endpoint
Status: in_progress
Depends on: US-007

Description:
As a content author, I want uploaded images enriched with AI-generated metadata so they are immediately descriptive and searchable.

Acceptance Criteria:
- [ ] server/src/services/enrich.js exports enrichAsset(db, id, openaiClient)
- [ ] enrichAsset reads asset row from DB by id, checks filepath is not null and file exists on disk
- [ ] Reads image file as Buffer, converts to base64 string
- [ ] Calls openaiClient.chat.completions.create with model: 'gpt-4o', messages: [{ role: 'user', content: [{ type: 'image_url', image_url: { url: 'data:<mime>;base64,<data>' } }, { type: 'text', text: '<prompt>' }] }]
- [ ] Prompt instructs model: 'Analyze this image and respond with ONLY valid JSON, no markdown: { "title": string, "description": string, "tags": string[], "location": string|null, "scene": string, "subjects": string[], "mood": string, "channel_hint": string, "confidence": number 0-1 }'
- [ ] Parses JSON from response, updates assets row: enriched_title, enriched_description, enriched_tags=JSON.stringify(tags), enriched_location, enriched_scene, enriched_subjects=JSON.stringify(subjects), enriched_mood, enriched_channel=channel_hint, enrichment_source='ai-vision', enrichment_confidence, updated_at=now
- [ ] POST /api/assets/:id/enrich route in server/src/routes/assets.js calls enrichAsset, returns 200 with updated full asset record
- [ ] OpenAI client initialized in server/src/index.js only if OPENAI_API_KEY is set, passed to routes via app.locals.openai
- [ ] If OPENAI_API_KEY not set: route returns 503 { error: 'AI enrichment unavailable — OPENAI_API_KEY not configured' }
- [ ] Jest test (jest.mock openai): enrichAsset maps all response fields correctly to DB columns
- [ ] Jest test (jest.mock openai): after enrich, asset has non-null enriched_title in DB
- [ ] Jest test (jest.mock openai): after enrich, FTS search on enriched_title returns the asset
- [ ] Negative case: asset has no filepath → 400 { error: 'No image file available for enrichment' }
- [ ] Negative case: filepath not found on disk → 404 { error: 'Image file not found on disk' }
- [ ] Negative case: OpenAI API throws → 502 { error: 'Enrichment failed', detail: message } — asset row unchanged
