### US-002: Share button — copy deep-link URL to asset
Status: in_progress
Depends on: None

Description:
As a content author, I want to copy a direct link to an asset so I can share it with a colleague who can open it immediately.

Acceptance Criteria:
- [ ] AssetDetailModal action row (next to Download button) renders a Share button
- [ ] Share button style: transparent background, border '1.5px solid #001B6B', color '#001B6B', border-radius 6px, padding '6px 14px', font-size 13px, label '↗ Share'
- [ ] On click: copies window.location.origin + '/?asset=' + encodeURIComponent(asset.id) to clipboard using navigator.clipboard.writeText
- [ ] After copy: button label swaps to '✓ Copied!' for 2000ms then reverts to '↗ Share'
- [ ] BrowsePage reads URLSearchParams('asset') on mount. If present and non-empty, sets selectedAssetId to that value
- [ ] After setting selectedAssetId from URL param, BrowsePage calls history.replaceState to remove the ?asset= param from the URL
- [ ] Example: user clicks Share on asset 'allure-aerial.jpg' → URL copied is 'http://localhost:5173/?asset=allure-aerial.jpg'
- [ ] Example: opening that URL in a new tab auto-opens the detail modal for that asset
- [ ] Negative case: if the ?asset= param contains an ID that doesn't exist, the modal opens and shows its loading/error state normally
- [ ] npm run lint passes
- [ ] npm run build passes
