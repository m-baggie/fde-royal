### US-003: Asset download from detail modal
Status: in_progress
Depends on: None

Description:
As a content author, I want to download the full-resolution asset directly from the detail modal so I can use it in my campaign without navigating elsewhere.

Acceptance Criteria:
- [ ] GET /api/assets/:id/download added to server/src/routes/assets.js
- [ ] Endpoint resolves the asset's web_image_path or thumbnail_path relative to DATA_DIR (same logic as the /media static route)
- [ ] Endpoint sets Content-Disposition: attachment; filename='<original filename>' and streams the file
- [ ] Returns 404 { error: 'Asset not found' } if asset ID does not exist
- [ ] Returns 404 { error: 'File not found' } if the image file is missing on disk
- [ ] AssetDetailModal right panel shows a Download button rendered as <a href='/api/assets/:id/download' download> with navy styling matching the Copy button
- [ ] Download button is always shown when asset is loaded (not conditional on CDN URL or enrichment)
- [ ] client/src/api/assets.js exports getAssetDownloadUrl(id) that returns the download URL string
- [ ] Supertest test: GET /api/assets/:id/download for a valid asset returns 200 with Content-Disposition header
- [ ] Supertest test: GET /api/assets/nonexistent/download returns 404
