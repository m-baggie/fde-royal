### US-003: XML ingest service — parse DAM export into SQLite
Status: in_progress
Depends on: US-002

Description:
As a developer, I want all 49 existing .content.xml files parsed and loaded into SQLite on first startup so assets are immediately available without a separate import step.

Acceptance Criteria:
- [ ] server/src/services/ingest.js exports ingestAssets(db, dataDir)
- [ ] Recursively walks dataDir using Node fs/path, finds .content.xml files where: parent folder name ends in .jpg/.jpeg/.png/.JPG AND full path does not contain '_jcr_content'
- [ ] Parses XML with Node's built-in DOMParser or the 'fast-xml-parser' package (npm install fast-xml-parser in server/) — reads all fields listed in dataModel
- [ ] Namespace attribute mapping: dc:title → original_title, dc:description → original_description, dc:creator → original_creator, dc:rights → original_dc_rights, xmpRights:Owner → original_rights_owner, xmpRights:UsageTerms → original_usage_terms, Iptc4xmpExt:LocationShown → original_location, cq:tags → original_tags, tiff:ImageWidth → width, tiff:ImageLength → height, dam:size → file_size, dam:Fileformat → file_format, dam:MIMEtype → mime_type, dam:scene7File → scene7_file, dam:scene7Domain → scene7_domain, dam:scene7FileStatus → scene7_status
- [ ] dam:lastS7SyncStatus is on the jcr:content element (not metadata child) — read it from there
- [ ] Strip JCR type wrappers from all values: '{Long}1920' → 1920, '{Boolean}true' → 'true', '{Date}2021-...' → '2021-...'
- [ ] Infer category/subcategory/subsubcategory from path segments relative to dataDir (last segment is the asset folder name itself — exclude it)
- [ ] Apply all deterministic rules from the rules section: creator normalization, rights reconciliation, channel/format inference, release placeholder detection, s7 sync error flag
- [ ] Set thumbnail_path to relative path of _jcr_content/renditions/cq5dam.thumbnail.319.319.png if it exists
- [ ] Set web_image_path to relative path of _jcr_content/renditions/cq5dam.web.1280.1280.jpeg if it exists
- [ ] Use INSERT OR IGNORE so re-running is safe
- [ ] server/src/index.js: after DB init, if SELECT COUNT(*) FROM assets = 0, call ingestAssets(db, process.env.DATA_DIR || '../Data/royal') and log 'Ingested N assets'
- [ ] Jest test: ingestAssets against the real ../Data/royal directory inserts exactly 49 rows
- [ ] Jest test: asset with id 'anthem-of-the-seas-new-york-statue-liberty.jpg' has original_title 'Anthem of the Seas Sightseeing the Statue of Liberty, New York'
- [ ] Jest test: creator '[Royal Carribean]' → enriched_creator_normalized = 'Royal Caribbean Brand Team'
- [ ] Jest test: exactly 7 assets have has_release_placeholder = 1
- [ ] Jest test: exactly 6 assets have s7_sync_error = 1
- [ ] Jest test: exactly 27 assets have enriched_rights_status = 'none'
- [ ] Jest test: asset 'anthem-of-the-seas-new-york-statue-liberty.jpg' has enriched_rights_status = 'owned'
- [ ] Negative case: a malformed XML file is skipped with console.warn — remaining files still ingest successfully
