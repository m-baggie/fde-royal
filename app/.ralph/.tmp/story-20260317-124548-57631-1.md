### US-001: Fix CDN URL double slash in assets route
Status: in_progress
Depends on: None

Description:
As a content author, I want the CDN URL in the asset detail to be a valid URL so I can copy and use it directly.

Acceptance Criteria:
- [ ] In server/src/routes/assets.js, update cdn_url construction to strip trailing slash from scene7_domain: `row.scene7_domain.replace(/\/+$/, '') + '/is/image/' + row.scene7_file`
- [ ] GET /api/assets/dubai-arabian-gulf-emirates-burj-al-arab-skyline.jpg returns cdn_url equal to 'https://assets.dm.rccl.com/is/image/RoyalCaribbeanCruises/dubai-arabian-gulf-emirates-burj-al-arab-skyline' (single slash after domain)
- [ ] Supertest test: cdn_url for any asset with scene7_domain set does not contain '//'
- [ ] Supertest test: cdn_url for an asset with no scene7_file is null
- [ ] Negative case: if scene7_domain has multiple trailing slashes (e.g. 'https://assets.dm.rccl.com///'), cdn_url still has exactly one slash before 'is/image'
