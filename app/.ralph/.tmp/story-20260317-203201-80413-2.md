### US-002: Smart upload — recursive folder and mixed drop support
Status: in_progress
Depends on: None

Description:
As a content author, I want to drop a folder of images onto the upload zone and have all images inside it queued automatically, without any prompts.

Acceptance Criteria:
- [ ] UploadModal drop zone handles onDrop with DataTransferItem.webkitGetAsEntry() to detect folders vs files
- [ ] A recursive async function traverseEntry(entry) collects all File objects from FileSystemFileEntry and FileSystemDirectoryEntry at any depth
- [ ] Only image files are collected: mime type starts with 'image/' OR extension is jpg/jpeg/png/gif/webp (case-insensitive)
- [ ] Non-image files are silently skipped — no error shown
- [ ] Dropping a mix of image files and a folder containing images queues all images from both sources
- [ ] The collected File array is passed to the existing upload queue logic unchanged
- [ ] Drop zone label updated to 'Drop images or folders here'
- [ ] Click-to-browse input: add a second input[type=file][webkitdirectory][multiple] button labelled 'Browse folder' alongside the existing 'Browse files' button, both feeding the same queue
- [ ] Vitest test: traverseEntry correctly filters out non-image files
- [ ] Vitest test: drop with a mix of file and folder entries queues only image files
