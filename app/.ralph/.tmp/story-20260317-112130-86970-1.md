### US-001: Scaffold monorepo — Express + React Vite
Status: in_progress
Depends on: None

Description:
As a developer, I want a working monorepo with Express backend and React Vite frontend so I can build and test both apps from a single root.

Acceptance Criteria:
- [ ] Root package.json has workspaces: ['client', 'server'] and name: 'dam-search'
- [ ] npm install concurrently at root
- [ ] Root scripts: dev (concurrently runs server dev + client dev), test (cd server && npm test && cd ../client && npm test), lint (cd server && npm run lint && cd ../client && npm run lint), build (cd client && npm run build)
- [ ] server/ package.json — install: npm install express better-sqlite3 multer cors dotenv openai uuid
- [ ] server/ devDependencies — install: npm install --save-dev jest supertest nodemon eslint
- [ ] server/ scripts: start (node src/index.js), dev (nodemon src/index.js), test (jest --runInBand --forceExit), lint (eslint src)
- [ ] server/src/index.js: creates Express app, uses cors({ origin: 'http://localhost:5173' }), uses express.json(), mounts GET /api/health → res.json({ status: 'ok' }), listens on PORT env var defaulting to 3001, logs 'Server running on port 3001'
- [ ] client/ created with: npm create vite@latest client -- --template react
- [ ] client/ install: npm install axios
- [ ] client/ devDependencies: npm install --save-dev vitest @testing-library/react @testing-library/jest-dom @testing-library/user-event jsdom eslint
- [ ] client/vite.config.js configures test: { environment: 'jsdom', globals: true, setupFiles: './src/test-setup.js' }
- [ ] client/src/test-setup.js imports @testing-library/jest-dom
- [ ] client/ scripts: dev (vite), build (vite build), test (vitest run), lint (eslint src)
- [ ] client/src/App.jsx renders <h1>Royal Caribbean DAM</h1>
- [ ] .env.example at project root contains: OPENAI_API_KEY=your_key_here, PORT=3001, DATA_DIR=../Data/royal
- [ ] .gitignore includes: node_modules/, dam.sqlite, uploads/, .env
- [ ] uploads/ directory created with a .gitkeep file
- [ ] Jest test: GET /api/health returns 200 { status: 'ok' }
- [ ] Vitest test: App renders text 'Royal Caribbean DAM'
- [ ] Example: npm run dev starts both servers without error
- [ ] Negative case: missing OPENAI_API_KEY does not crash server — logs warning 'AI enrichment disabled: OPENAI_API_KEY not set' and continues
