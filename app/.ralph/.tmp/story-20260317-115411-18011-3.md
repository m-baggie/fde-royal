### US-011: Asset detail modal
Status: in_progress
Depends on: US-010

Description:
As a content author, I want a full-detail popup for any asset so I can compare original vs enriched metadata, copy the CDN URL, and trigger AI enrichment.

Acceptance Criteria:
- [ ] client/src/components/AssetDetailModal.jsx renders as a fixed full-screen overlay when selectedAssetId is non-null
- [ ] Modal fetches full asset via getAsset(id) on open — shows loading spinner until resolved
- [ ] Overlay: fixed, inset 0, z-index 1000, background rgba(0,0,0,0.6). Inner panel: white, border-radius 12px, max-width 1100px, 90vw, max-height 90vh, overflow-y auto, centred with flex
- [ ] Close button × at top-right — closes modal. ESC keydown listener added on mount, removed on unmount
- [ ] body overflow:hidden added when modal opens, restored on close
- [ ] Left panel (40% width): image tag src=web_image_path or thumbnail_path (proxied through Express /thumbnails/), grey placeholder if both null. Below image: filename in bold, 'W × H px' dimensions, formatted file size, format badge
- [ ] Right panel (60% width): scrollable. Metadata table — two columns with headers 'Original' and 'Enriched' in navy. Rows: Title, Description, Tags, Location, Creator, Rights Owner, Usage Terms, Channel, Format. Cell values: enriched column shows enriched_* value in navy; null shows grey '—'. If enriched value is non-null AND differs from original: add 3px left border gold on enriched cell
- [ ] Quality Issues section below table: only renders if quality_issues.length > 0. Each issue shown as a badge: 'release_placeholder' and 'missing_rights' → red badge; others → amber badge. Badge labels: missing_title='Missing Title', missing_rights='No Rights Data', release_placeholder='Release Placeholder — Review Required', s7_sync_error='CDN Sync Error', etc.
- [ ] CDN URL section: renders only if cdn_url is non-null. Shows label 'CDN URL', full url in <code> monospace block, Copy button. On Copy click: navigator.clipboard.writeText(cdn_url), button text → 'Copied!' for 2000ms then reverts to 'Copy'
- [ ] 'Enrich with AI' button shown only if enrichment_source is null or 'pending'. On click: calls enrichAsset(id), button shows spinner + 'Enriching...', on success re-fetches asset and refreshes modal, on 503 shows toast 'Enrichment unavailable — add OPENAI_API_KEY to .env'
- [ ] Vitest test: modal renders display_title from mocked getAsset response
- [ ] Vitest test: ESC keydown triggers onClose
- [ ] Vitest test: clicking × button triggers onClose
- [ ] Vitest test: Copy button shows 'Copied!' immediately after click (mock clipboard API)
- [ ] Vitest test: 'Enrich with AI' button is visible when enrichment_source is null
- [ ] Vitest test: quality_issues badge 'No Rights Data' renders when 'missing_rights' in quality_issues
- [ ] Negative case: cdn_url is null → CDN URL section is not rendered
- [ ] Negative case: enrichAsset returns 503 → toast shown, modal remains open
