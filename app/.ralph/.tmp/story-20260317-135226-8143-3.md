### US-003: Add Channel and Scene filter sections to FilterSidebar
Status: in_progress
Depends on: US-002

Description:
As a content author, I want to filter assets by AI-inferred channel and scene so I can quickly find the right format for my campaign.

Acceptance Criteria:
- [ ] FilterSidebar renders a 'Channel' collapsible section below existing sections — only rendered if filters.channels is non-empty
- [ ] Channel chips: one chip per value in filters.channels — clicking selects/deselects (single-select, same as category)
- [ ] FilterSidebar renders a 'Scene / Mood' collapsible section — only rendered if filters.scenes is non-empty
- [ ] Scene chips: multi-select chips from filters.scenes — clicking toggles inclusion
- [ ] Both sections start collapsed by default
- [ ] Active channel selection shown as a removable chip in the active filters row below the search bar
- [ ] Active scene selections shown as removable chips in the active filters row
- [ ] Clearing a chip deselects the filter and re-fetches assets
- [ ] getAssets() call in BrowsePage includes channel and scene params when set
- [ ] Vitest test: FilterSidebar with channels=['hero','banner'] renders a 'Channel' section with two chips
- [ ] Vitest test: FilterSidebar with channels=[] does not render a 'Channel' section
- [ ] Vitest test: clicking a channel chip calls onFilterChange with key='channel' and the chip value
- [ ] Negative case: if API returns no channels and no scenes, FilterSidebar renders identically to before enrichment (no empty sections)
