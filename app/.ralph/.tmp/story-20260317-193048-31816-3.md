### US-002: API — grouped browse and variants endpoint
Status: in_progress
Depends on: US-001

Description:
As a developer, I want the search API to default to grouped mode and expose a variants endpoint so the UI can show grouped cards and a variant switcher.

Acceptance Criteria:
- [ ] GET /api/assets default: WHERE (is_primary_variant = 1 OR variant_group_id IS NULL)
- [ ] Each asset in response includes variant_count (integer, minimum 1)
- [ ] GET /api/assets?show_all_variants=1 returns all assets with no grouping filter
- [ ] GET /api/assets/:id/variants returns array of all assets in the same group, empty array if no group
- [ ] client/src/api/assets.js exports getAssetVariants(id) that calls GET /api/assets/:id/variants
- [ ] Supertest test: GET /api/assets returns asset with variant_count=4 for the galveston primary
- [ ] Supertest test: GET /api/assets/:id/variants for a galveston asset returns 4 items
- [ ] Supertest test: GET /api/assets/:id/variants for an ungrouped asset returns []
