### US-006: Sidebar filter visual restyle
Status: in_progress
Depends on: US-001, US-002

Description:
As a user, I want the sidebar filters to feel refined and consistent with the rest of the redesigned UI so it doesn't look like a legacy widget stuck on the left.

Acceptance Criteria:
- [ ] Sidebar container: background #F7F8FB (from US-001), padding 16px 12px, no right border (remove the dividing border — let the background color difference create separation)
- [ ] Section divider: change from bottom border to 8px margin gap between sections — lighter visual separation
- [ ] Section header chevron (▲/▼): replaced with a simpler − / + indicator, font-size 10px, color #9CA3AF
- [ ] Chip resting state: background #FFFFFF, border '1px solid #E5E7EB', border-radius 8px, color #374151, font-size 12px, font-weight 500, padding 4px 10px
- [ ] Chip hover: background #F3F4F6, border-color #D1D5DB, transition 120ms ease
- [ ] Chip active: background #001B6B, border-color #001B6B, color #FFFFFF
- [ ] Location dropdown select: border-radius 8px, border '1px solid #E5E7EB', font-size 13px, padding 7px 10px, background #FFFFFF
- [ ] Example: an active 'ships' chip shows navy background and white text; an inactive chip shows white background with grey border
- [ ] Negative case: section with no options (empty array) renders nothing — no empty chip row or blank space
