### US-001: useFavourites hook — localStorage state management
Status: in_progress
Depends on: None

Description:
As a developer, I want a custom hook that manages favourites state in localStorage so all components share a single source of truth.

Acceptance Criteria:
- [ ] client/src/hooks/useFavourites.js exports useFavourites() hook
- [ ] Hook returns { favouriteIds: string[], isFavourited(id): boolean, toggle(id, assetData): void, clear(): void, count: number }
- [ ] Initialises favouriteIds from localStorage key 'dam_favourites' on mount (JSON.parse, default [])
- [ ] toggle(id, assetData): if id in favouriteIds → remove it and delete from 'dam_favourites_data'; if not → add it and write { display_title, thumbnail_path, cdn_url } to 'dam_favourites_data'
- [ ] Every state change writes updated array to localStorage 'dam_favourites'
- [ ] clear(): empties favouriteIds, removes both localStorage keys
- [ ] App.jsx calls useFavourites() and passes { favouriteIds, isFavourited, toggle: onFavouriteToggle, clear, count } as props to BrowsePage
- [ ] Example: toggle('asset-1', data) → localStorage 'dam_favourites' contains ['asset-1']
- [ ] Negative case: toggling the same ID twice returns to empty array
- [ ] npm run lint passes
- [ ] npm run build passes
