### US-005: Search bar refinement
Status: in_progress
Depends on: US-001

Description:
As a user, I want the search bar to feel more intentional and premium so the primary action of the page gets appropriate visual weight.

Acceptance Criteria:
- [ ] SearchBar.jsx input element: height 52px (was shorter), border-radius 10px, border '1.5px solid #E5E7EB' at rest
- [ ] On focus: border-color changes to #001B6B, box-shadow '0 0 0 3px rgba(0,27,107,0.08)', background #FFFFFF, transition 150ms ease
- [ ] A search icon (🔍 or SVG magnifier) is rendered inside the input on the left side, inset 14px from left edge, color #9CA3AF at rest → #001B6B on focus
- [ ] Input left padding increased to 44px to accommodate the icon (was ~12px)
- [ ] Placeholder text color: #9CA3AF, font-style normal (not italic)
- [ ] The search bar container row in BrowsePage.jsx: padding updated to 10px 24px to accommodate the taller input
- [ ] Example: clicking the search bar shows a navy border glow and the search icon turns navy
- [ ] Negative case: unfocused search bar with no text shows grey border and grey icon — no navy color applied
- [ ] npm run lint passes
- [ ] npm run build passes
