### US-001: Admin mode toggle — localStorage persistence + gate Upload and Delete
Status: in_progress
Depends on: None

Description:
As an admin user, I want to toggle admin mode from the sidebar so that upload and delete actions are only available when I need them.

Acceptance Criteria:
- [ ] App.jsx holds adminMode state (boolean, default false) initialised from localStorage key 'dam_admin_mode'
- [ ] App.jsx passes adminMode and onAdminModeChange to BrowsePage which passes them to FilterSidebar and AssetDetailModal
- [ ] FilterSidebar renders an admin toggle at the bottom, separated by a top border, with label 'Admin Mode' and a pill-style checkbox toggle
- [ ] Toggle active state: navy (#001B6B) background track, white thumb. Inactive: #D1D5DB track
- [ ] Clicking the toggle calls onAdminModeChange, updates App state, and writes '1' or removes the key from localStorage
- [ ] Header Upload button: only rendered when adminMode is true
- [ ] AssetDetailModal Delete button: only rendered when adminMode is true
- [ ] Example: page loads with no localStorage key → Upload button absent, Delete button absent
- [ ] Example: toggle admin mode on → Upload and Delete appear immediately without page refresh
- [ ] Negative case: refreshing the page with admin mode on keeps admin mode on (localStorage persists)
- [ ] npm run lint passes
- [ ] npm run build passes
