### US-001: POST /api/search/prompt — Claude-powered search endpoint
Status: in_progress
Depends on: None

Description:
As a developer, I want a dedicated smart search endpoint so Claude can interpret natural language prompts and return structured search results.

Acceptance Criteria:
- [ ] POST /api/search/prompt added to server/src/routes/search.js (new file) and mounted at /api/search in server/src/index.js
- [ ] Accepts JSON body { prompt: string }, returns 400 if prompt is missing or empty
- [ ] Calls Claude Haiku with the RC-specific extraction prompt and parses the JSON response
- [ ] On success: calls searchAssets() with extracted search_terms (joined with OR), plus any non-null filter values (content_type, mood, location, destination_region, channel)
- [ ] Returns { total, assets, explanation } — explanation is Claude's plain-English summary string
- [ ] On Claude failure or invalid JSON: falls back to searchAssets({ q: prompt }) and returns explanation: null
- [ ] If ANTHROPIC_API_KEY is not configured: falls back to plain search, returns explanation: null
- [ ] Supertest test: POST /api/search/prompt with { prompt: 'ships at sunset' } returns 200 with assets array and total
- [ ] Supertest test: POST /api/search/prompt with empty body returns 400
- [ ] Supertest test: endpoint works when anthropic is null (falls back gracefully)
- [ ] client/src/api/assets.js exports smartSearch(prompt) that POSTs to /api/search/prompt and returns { total, assets, explanation }
