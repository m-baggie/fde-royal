### US-001: Semantic query expansion via Claude Haiku
Status: in_progress
Depends on: None

Description:
As a content author, I want to search with natural language like 'happy family on deck' and get relevant results even when those exact words aren't in the metadata.

Acceptance Criteria:
- [ ] GET /api/assets route is made async
- [ ] An expandQuery(anthropic, q) async function is added in routes/assets.js
- [ ] expandQuery calls claude-haiku-4-5-20251001 with the RC-specific prompt and returns original q joined with expanded terms using FTS5 OR syntax
- [ ] If anthropic is null/undefined, expandQuery returns q unchanged
- [ ] If the Claude call throws or returns invalid JSON, expandQuery catches the error and returns original q unchanged
- [ ] The expanded query string is passed to searchAssets() in place of the original q
- [ ] Supertest test: GET /api/assets?q=sunset returns results (expansion is mocked or skipped in tests)
- [ ] Supertest test: search still works when ANTHROPIC_API_KEY is not set (falls back to plain FTS5)
