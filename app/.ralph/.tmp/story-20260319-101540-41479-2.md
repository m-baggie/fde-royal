### US-002: BrowsePage — split-screen layout with slide-in panel
Status: in_progress
Depends on: US-001

Description:
As a content author, I want the detail panel to appear alongside the grid so I can keep browsing while reading asset details.

Acceptance Criteria:
- [ ] BrowsePage removes the AssetDetailModal import and renders AssetDetailPanel instead
- [ ] AssetDetailPanel is rendered inside the bodyRow flex container as a direct sibling of the main content div, not as a fixed overlay
- [ ] When selectedAssetId is null: panel is not rendered, grid takes full available width
- [ ] When selectedAssetId is set: panel renders at 400px width, grid shrinks to fill remaining space (flex:1, min-width:0)
- [ ] Panel slide-in animation: panel mounts with transform translateX(400px) then immediately transitions to translateX(0) over 250ms with cubic-bezier(0.16,1,0.3,1) easing
- [ ] Panel slide-out: on close, transitions back to translateX(400px) over 200ms then unmounts (use a 'isClosing' state)
- [ ] Clicking a different card while panel is open: selectedAssetId updates, panel stays open and shows new asset (no close/reopen)
- [ ] AssetDetailModal.jsx can be deleted or kept but is no longer used in BrowsePage
- [ ] The grid remains scrollable and cards remain clickable while the panel is open
- [ ] Example: grid shows 4 columns with panel closed; with panel open grid shows ~2-3 columns depending on viewport
- [ ] Example: clicking card A opens panel showing asset A; clicking card B updates panel to show asset B without animation
- [ ] Negative case: pressing Escape while panel is open closes the panel
- [ ] npm run lint passes
- [ ] npm run build passes
